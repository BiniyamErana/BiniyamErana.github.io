1. Rewrite the JSP page in the first demo (forEach) to use scripting instead of the JSTL forEach.
Answer: <% Student[] table = (Student[]) request.getAttribute("students"); for (Student stu: table) { %> <%= stu.getName() %> <%= stu.getAge() %> <% } %>

2. How do custom tags relate to JSTL?
Answer: They are all JSP actions. Standard actions are part of any servlet web container. JSTL is a separate library that needs to be added to an application.

3. What is the role of the URI in the TLD and the taglib directive?
Answer: It is a unique identifier for a set of custom tags. It allows the container to find the TLD for the tags in that taglib directive.

4. What is a tag handler class?
Answer: Java class that extends SimpleTagSupport. Has a doTag method to generate the markup for the tag, and setters for any attributes in the tag. Tag developer writes this class.

5. What is the role of attribute setters in a tag handler class?
Answer: The container will call the setters to give the tag handler object the values from the tag attributes so that the doTag method can use these values when generating the markup.

6. What is the role of the doTag() method in a tag handler class?
Answer: This is the method that generates the markup for the tag.

7. What does the operation getJspContext().getOut().write(“Hi Bob”) do when called in a doTag() method?
Answer: It will write "Hi Bob" to the output stream that this tag is writing to.

8. What does the operation getJspBody().invoke(null) do when called in a doTag() method?
Answer: Processes the body of the tag—meaning that any template text will be sent to the output stream and any JSP scripting or EL will be evaluated and included in the output text.
