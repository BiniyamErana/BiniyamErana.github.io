Servlet Intro Quiz

Home

1. What is the difference between a web server and a web container? Answer: Web server is hardware prepared for clients to interact with the whole system while web container is solely responsible for connecting and processing client side information with knowing the outside sources, such as database and other applications.

2. What is a servlet? Answer: Servlet is a class for handling a request from the client side.

3. How do web servers and web containers interact with servlets? Answer: creating servlet object and using methods such as init, service, destroy...

4. Who creates request objects? Answer: Client side creates the request object.

5. What are the states in the servlet lifecycle? Answer: init, service, destroy

6. Who calls init and when? Answer: Web container calls init when servlet object first created.

7. Which of init, service, and doGet should you override? Answer: doGet

8. In what sense are servlets multi-threaded? Answer: In sense of calling "service" method, it is multi-threaded.

9. What are the implications of this for servlet instance variables? Answer: Instance variables can be shared concurrently.
